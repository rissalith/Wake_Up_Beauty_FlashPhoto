# 职业照和证件照提示词优化说明

## 问题背景
用户反馈生成的职业照和证件照头身比例不对，需要优化提示词以确保更准确的构图。

## 优化内容

### 1. 证件照提示词优化

**原提示词问题：**
- 缺少详细的头身比例要求
- 没有明确的构图空间分配
- 负面提示词不够具体

**新提示词特点：**
```
基于参考图生成专业{{spec}}证件照。【核心要求】必须严格保持参考图中人物的面部特征一致性，
包括五官轮廓、脸型、眼睛形状、鼻子、嘴巴、肤色等，确保生成结果与本人高度相似。

纵向竖版构图（高度大于宽度，宽高比约3:4），{{background}}。

正面免冠，光线均匀，人物居中，采用标准证件照构图：
- 头顶到画面顶端留有约8%-10%空间
- 头部（从下巴到头顶）占画面高度约55%-60%
- 肩部需完整露出至胸口上方位置

【重要】必须保持人物主体完整，肩膀两侧不能被画面边缘裁切。
```

**改进点：**
- ✅ 明确头身比例：头部占55%-60%
- ✅ 明确头顶空间：8%-10%
- ✅ 强调肩膀完整性
- ✅ 增强面部特征保持要求
- ✅ 改进负面提示词：避免头部过小/过大、肩膀被裁切

### 2. 职业照提示词优化

**原提示词问题：**
- 缺少具体的头身比例要求
- 没有明确的构图指导
- 面部特征保持要求不明确

**新提示词特点：**
```
基于参考图生成专业职业照。【核心要求】必须严格保持参考图中人物的面部特征一致性，
包括五官轮廓、脸型、眼睛形状、鼻子、嘴巴、肤色等，确保生成结果与本人高度相似。

半身竖版构图（高度大于宽度，宽高比约3:4），{{background}}，{{style}}。

正面或微侧面，自信表情，光线均匀，人物居中，采用标准职业照构图：
- 头顶到画面顶端留有约5%-8%空间
- 头部（从下巴到头顶）占画面高度约40%-45%
- 肩膀需完整露出至胸口上方位置

【重要】必须保持人物主体完整，肩膀两侧不能被画面边缘裁切。
```

**改进点：**
- ✅ 明确头身比例：头部占40%-45%（比证件照更小，更显身材）
- ✅ 明确头顶空间：5%-8%（比证件照更紧凑）
- ✅ 允许微侧面和自信表情（更符合职业照风格）
- ✅ 强调肩膀完整性
- ✅ 增强面部特征保持要求
- ✅ 改进负面提示词

### 3. 关键改进对比

| 指标 | 证件照 | 职业照 | 说明 |
|------|-------|--------|------|
| 头身比例 | 55%-60% | 40%-45% | 职业照更显身材 |
| 头顶空间 | 8%-10% | 5%-8% | 职业照更紧凑 |
| 面部角度 | 正面免冠 | 正面或微侧面 | 职业照更灵活 |
| 表情 | 中性表情 | 自信表情 | 职业照更专业 |
| 宽高比 | 3:4 | 3:4 | 都是竖版 |

## 部署方式

### 方式一：自动迁移（推荐）
```bash
cd core-api
node migrations/run-optimize-prompts.js
```

### 方式二：手动执行 SQL
```bash
sqlite3 data/flashphoto.db < migrations/optimize-portrait-prompts.sql
```

### 方式三：Docker 部署时自动执行
迁移脚本会在 Docker 容器启动时自动运行。

## 验证方法

1. **查看数据库更新：**
```bash
sqlite3 core-api/data/flashphoto.db \
  "SELECT name, quality_score FROM knowledge_base WHERE category='scene_template' AND name LIKE '%照%';"
```

2. **测试生成：**
   - 上传照片到证件照场景
   - 观察生成结果的头身比例是否符合预期
   - 检查肩膀是否完整

3. **对比效果：**
   - 新提示词应该生成更规范的头身比例
   - 肩膀不应该被裁切
   - 面部特征应该与原图高度相似
